# UI_FLOW: 화면 구성 & 디자인 가이드

> 와이어프레임을 텍스트로 명시화한 문서.  
> Cursor에게 화면을 구현시킬 때 해당 섹션을 통째로 참조시켜라.

---

## 1. 디자인 원칙

### 1.1 핵심 컨셉

> **"긴장감 있는 금융 앱"**

유가 급등 시대라는 맥락을 반영해, 밝고 귀여운 톤이 아닌 **신뢰감 있고 실용적인** 금융 앱 톤으로 간다. 토스, 뱅크샐러드, Webull이 레퍼런스.

### 1.2 디자인 원칙 5가지

1. **가격은 크게, 나머지는 조용히** — 가격이 항상 화면의 주인공
2. **3초 룰** — 앱 진입 후 3초 안에 답을 찾을 수 있다
3. **한 손 조작** — 주요 CTA는 화면 하단 60% 영역에 배치
4. **상태를 속이지 않는다** — 오프라인이면 명시, 오래된 데이터면 명시
5. **컬러는 의미 있게** — 장식성 색상 금지, 모든 색상은 상태를 나타낸다

## 4. 화면별 상세 설계

### 4.1 스플래시

**요소:**
- 중앙에 로고 + 앱 이름
- 하단에 로딩 인디케이터
- 배경: 그라디언트 (bgPrimary → bgSecondary)

**동작:**
- Hive 초기화, 위치 권한 체크, 온보딩 여부 확인
- 최소 1초 표시 (깜빡임 방지)

---

### 4.2 온보딩

**3단계 페이지뷰:**

**Page 1: "지금은 유가 비상 시국"**
- 일러스트: 주유소 + 상승 화살표
- 헤드라인: "L당 2,000원 시대"
- 서브: "우리 동네 가장 싼 주유소를 찾아드려요"

**Page 2: "정부 최고가격제 참고"**
- 일러스트: 정부 배지 + 가격표
- 헤드라인: "정부 최고가와 비교"
- 서브: "합리적인 주유 결정을 도와드려요"

**Page 3: "주유 기록 자동 관리"**
- 일러스트: 가계부 + 차트
- 헤드라인: "주유 기록, 한 번만 입력"
- 서브: "리터, 단가, 연비까지 자동 계산"
- **CTA:** [시작하기] → 위치 권한 요청

**공통 요소:**
- 우상단 [건너뛰기]
- 하단 페이지 인디케이터 (3점)

## 6. 애니메이션 가이드

| 요소 | 애니메이션 | 지속 시간 |
|---|---|---|
| 화면 전환 | 슬라이드 (왼→오) | 300ms |
| 바텀시트 | 이지 아웃 스프링 | 400ms |
| 가격 변동 숫자 | Count-up 애니메이션 | 600ms |
| 마커 탭 | 스케일 1.0 → 1.2 → 1.0 | 200ms |
| 로딩 | Shimmer (카드 형태 유지) | 1500ms loop |

## 7. 접근성 체크리스트

- [ ] 모든 텍스트 최소 14sp, 가격은 20sp 이상
- [ ] 컬러만으로 정보 전달 금지 (아이콘/텍스트 병행)
- [ ] 최소 터치 영역 48x48 dp
- [ ] 폼 입력 필드에 라벨 + placeholder 분리
- [ ] 주요 화면에 Semantics 위젯 적용 (스크린 리더)
- [ ] 시스템 다크모드 연동 (강제 다크모드 아닌 선택)

- 개별 주유소 탭 → 해당 상세 화면으로 딥링크
